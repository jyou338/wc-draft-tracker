# World Cup Draft Table — project notes

A standings site for a 12-manager fantasy draft built on the 2026 FIFA World Cup.
Each manager owns one nation; points come from that nation's match events. Static
Next.js so it runs free on Vercel's Hobby plan — no database, no backend.

## Stack
- Next.js 14 (App Router), React 18, TypeScript
- Plain CSS (`app/globals.css`), no Tailwind — keeps the build dependency-free
- Google Fonts (Oswald / Inter / JetBrains Mono) via a `<link>` in `app/layout.tsx`
  (loaded at runtime, so the build needs no network)
- Mostly static (`○ (Static)`); client components only where interactivity is needed

## Commands
- `pnpm install` then `pnpm dev` → http://localhost:3000
- `pnpm build` to verify before pushing
- `node scripts/fetch-wc-data.mjs` — refresh data from ESPN (no API key needed)
- Deploy: push to GitHub → Vercel auto-redeploys

## Where things live
- `data/tournament.ts` — THE data file. Auto-generated by the fetch script.
  Draft picks, completed `results`, upcoming `fixtures`, live `liveMatches`, and `lastUpdated`.
- `lib/scoring.ts` — point weights + which are still assumptions
- `lib/standings.ts` — computes per-manager totals and ranks from results × weights
- `components/Scoreboard.tsx` — server component; shows standings with LIVE badge + top scorers
- `components/MatchLog.tsx` — **client component**; matchday tabs + results cards
- `components/MatchHistory.tsx` — client component; collapsible per-team match history
- `components/Fixtures.tsx` — server component; upcoming matches with local kick-off times
- `components/PointsChart.tsx` — **client component**; SVG line chart with hover tooltips
- `components/LocalTime.tsx` — client component; converts ISO kick-off times to browser timezone
- `app/page.tsx` — composes the page
- `scripts/fetch-wc-data.mjs` — fetches ESPN API and rewrites `data/tournament.ts`
- `.github/workflows/update-data.yml` — smart cron: every 10 min during match hours, hourly overnight

## Data pipeline
1. GitHub Actions cron runs every 10 min (12:00–23:59 UTC) or hourly overnight
2. `scripts/fetch-wc-data.mjs` fetches ESPN's public scoreboard API (no key required),
   looping through all WC dates (Jun 11–Jul 19), and rewrites `data/tournament.ts`
3. If the file changed, the bot commits and pushes
4. Vercel detects the push and rebuilds the static site (~30s)
5. Users see fresh data — no API calls happen at page load time

## Data model (`data/tournament.ts`)
```ts
interface Result {
  matchday: number;
  team: string;
  opponent: string;
  scoreFor: number;
  scoreAgainst: number;
  goals: string[];       // named per player — from ESPN details[]
  assists: number;       // team total count only — ESPN doesn't expose individual assisters
  cleanSheet: boolean;   // derived from scoreAgainst === 0
  yellowCards: string[]; // named per player — from ESPN details[]
  redCards: string[];    // named per player — from ESPN details[]
  ownGoals: string[];    // named per player — from ESPN details[]
}

interface Fixture {
  matchday: number;
  team: string;
  opponent: string;
  kickoff?: string;    // formatted UTC string e.g. "Sat 00:30 UTC"
  kickoffISO?: string; // raw ISO date e.g. "2026-06-20T00:30Z" — used by LocalTime
}

interface LiveMatch {
  team: string;
  opponent: string;
  scoreFor: number;
  scoreAgainst: number;
  minute: string; // display clock e.g. "67'"
}
```

## ESPN API
- Scoreboard: `https://site.api.espn.com/apis/site/v2/sports/soccer/fifa.world/scoreboard?dates=YYYYMMDD`
- `competitions[].details[]` — goals, cards, own goals with named player per event
- `competitors[].statistics` — includes `goalAssists` count (team total, no player name)
- `competition.status.type.state` — `"pre"` / `"in"` / `"post"` for live detection
- `competition.status.displayClock` — current match minute for live matches
- **Assists are not available per-player** from any ESPN endpoint — team count only

## Scoring (`lib/scoring.ts`)
- **Confirmed by the organiser:** goal **+5**, assist **+2**
- **Assumed (not yet confirmed):** clean sheet +3, yellow −1, red −5, own goal −10.
  These are flagged with `*` in the footer. Change a number, redeploy, totals recompute.

## UI features
- **Scoreboard**: rank · `OWNER · TEAM` title · next fixture (local time) · top goal scorers · points
- **LIVE badge**: pulsing green pill on scoreboard row when team is mid-game; shows score + minute
- **Points Race chart**: SVG line chart, cumulative points per manager per matchday.
  Hover dots/lines to see name tooltip; overlapping dots (same pts) all highlight together.
- **Match log**: matchday tabs (All / MD1 / MD2 …); defaults to latest matchday
- **Fixtures**: kick-off times shown in the visitor's local timezone via `LocalTime` component.
  Falls back to UTC on SSR to avoid hydration mismatch; switches to local time after mount.
  Timezone abbreviation derived from long English name (e.g. "New Zealand Standard Time" → "NZST").

## Conventions
- Keep it static and free — don't add a database or server unless explicitly asked.
- `data/tournament.ts` is auto-generated — don't edit by hand, run the script instead.
- Manager names display in uppercase via CSS; flags are emoji on each `DraftPick`.
- Client components are only used where needed (interactivity / browser APIs).
